x264.py

x264.py is designed to make it easier to pipe input from ffmpeg into x264. This is typically useful if the input must be processed in a 32bit process, but the output should be encoded using a 64bit application. (see: AviSynth)

x264.py cuts out the work of detecting the resolution, colorspace, frame rate, and frame count of the input video, along with the setup of the piped commands. Typically, it's as easy as replacing the x264 part of the command line with x264.py.

Required software

  • python – versions 2.x or 3.x should both work
  • ffmpeg & ffprobe – for video info and piping
  • x264 – for video encoding

Download

  • x264.py / download – main python application

Script Arguments

The following describes some of the flags that can be used where <script-arguments> appears in the next section. Additional info about these and other flags can be found by executing x264.py --usage.

  • --ffmpeg <exe_path> – Path to the executable ffmpeg file
    Defaults to "ffmpeg"; if ffmpeg is in the PATH environment variable, it will automatically work.
    This typically points to a 32bit version of ffmpeg.
  • --ffprobe <exe_path> – Path to the executable ffprobe file
    Defaults to "ffprobe"; if ffprobe is in the PATH environment variable, it will automatically work.
    This typically points to a 32bit version of ffprobe, but there are circumstances under which a 64bit version might be used.
  • --x264 <exe_path> – Path to the executable x264 file
    Defaults to "x264"; if x264 is in the PATH environment variable, it will automatically work.
    This typically points to a 64bit version of x264.
  • --no-command | --no-ffmpeg-stderr | --no-time | --no-return-codes – Suppress a certain type of output
    These 4 flags specify that a certain type of output should be omitted. If all 4 flags are included, the only output that will display is the output from the x264 process.

Usage

Examples following will use this command as the base command to be modified:

x264 --crf 18 --preset veryslow --output video.mkv input.avs
Note that it is important that the input file (input.avs) appears as the last argument in each example!

  • x264.py --crf 18 --preset veryslow --output video.mkv input.avs
    This example runs it with x264.py, piping the video from ffmpeg into x264 before outputting the video.
  • x264.py <script-arguments> @ --crf 18 --preset veryslow --output video.mkv input.avs
    This example shows the setup for specifying additional settings for x264.py.
    The @ signals the end of the script-arguments.
  • x264.py
    --ffmpeg ffmpeg-x86.exe --ffprobe ffprobe-x86.exe --x264 x264-10b-x64.exe @
    --crf 18 --preset veryslow --output video.mkv input.avs
    This example directly specifies the executables: 32bit ffmpeg and ffprobe; 64bit x264 with 10bit color output
